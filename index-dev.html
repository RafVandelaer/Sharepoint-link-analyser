Quick Guide

Start with a quick overview, create the Entra app, run the script, and then upload the generated CSV.
1. What the script does

The script scans SharePoint Online and collects all sharing links found in document libraries.

  • Connects interactively using your Microsoft account
  • Temporarily grants Site Collection Admin access when required
  • Scans files and folders for sharing links
  • Exports all results to a CSV file

After the scan, you can upload the CSV to analyze:

  • External sharing
  • Anonymous links
  • Expired links
  • Risky permissions

All processing happens locally. No SharePoint data is stored or transmitted.

2. One-time setup

On the first run, use OnMicrosoftDomain so the script can create and store the Entra app ClientId for this tenant.

pwsh .\getallsharinglinks-full.ps1 `
-TenantName yourtenant `
-OnMicrosoftDomain yourtenant `
-OutputDirectory .\output

TenantName is the SharePoint tenant prefix used in the admin URL.

https://yourtenant-admin.sharepoint.com

In this case:

TenantName = yourtenant

After the app is created, the script asks whether it should start the scan immediately or stop and show you the rerun command. You normally only do this setup once per tenant.

3. Run a scan

After the first setup, the stored ClientId is reused automatically. In normal use, this command is enough:

pwsh .\getallsharinglinks-full.ps1 `
-TenantName yourtenant `
-OutputDirectory .\output

Sign in when prompted and wait for the scan to finish. The script generates a file like:

SharingLinks-<tenant>-<timestamp>.csv

During the scan, the script connects to the SharePoint admin site, restores any interrupted cleanup if needed, and then processes sites, libraries, files, and folders one by one.

  • Shows progress while counting sites, libraries, items, files, folders, and sharing links
  • Temporarily adds Site Collection Admin access when required and removes it again afterwards
  • Writes the main CSV plus recovery and log files in the selected output folder

Besides the CSV, the script also creates a state file, event log, cleanup queue, and summary file for recovery and audit purposes.

If you run the script on another machine or no stored ClientId is found, you can still pass it explicitly with -ClientId "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx".

Upload this CSV file to analyze the results.

4. Optional parameters

You can limit the scan scope if needed.

Scan a single site
-SiteUrl https://tenant.sharepoint.com/sites/project
Include personal OneDrive sites
-IncludeOneDrive
Limit scan size (for testing)
-MaxSites 5
-MaxLibrariesPerSite 5
-MaxItemsPerLibrary 500
Cleanup after interrupted scan
-RecoveryOnly
Example
pwsh .\getallsharinglinks-full.ps1 -TenantName contoso -SiteUrl https://contoso.sharepoint.com/sites/project -MaxSites 5
