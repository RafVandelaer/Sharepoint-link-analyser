Quick Guide

Start with a quick overview, create your Entra app, run the script, and then upload the generated CSV here.
1. What the script does

The script signs in interactively and scans the SharePoint sites you selected.

It checks document libraries, looks at files and folders, and collects the sharing links it finds into one CSV export.

If temporary extra access is needed to complete the scan, the script adds that access, logs it, and removes it again afterwards.

This page then turns that CSV into summaries, charts, and a browsable tree view so you can review the results more easily.

We do not store, keep, or analyze your CSV data on a server. Everything here runs locally in your browser with privacy in mind.

2. Create one Entra App Registration

Go to Entra Admin Center.

Open App registrations and choose New registration.

Use these settings:
Name: PnP PowerShell Login
Supported account types: Accounts in any organizational directory (Multitenant)
Redirect URI: Public client / native with http://localhost

Then open the new app registration and go to Authentication.

Set Allow public client flows to Yes.

Finally, go to Overview and copy the Application (client) ID.

You will use it like this:
-ClientId "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"

3. Run the script

Download `getallsharinglinks-full.ps1`, open PowerShell 7, move to the folder containing the script, and run for example:

pwsh ./getallsharinglinks-full.ps1 -TenantName yourtenant -ClientId "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx" -OutputDirectory ./output

`yourtenant` is often the tenant prefix from `yourtenant.onmicrosoft.com` and also the part before `-admin.sharepoint.com`. Sign in through the interactive prompt, let the scan finish, then upload the generated `SharingLinks-*.csv` in this page.

4. Optional parameters
-SiteUrl Scan only one specific SharePoint site instead of the broader tenant scope.
-IncludeOneDrive Include personal OneDrive sites in the scan.
-MaxSites Limit how many sites are processed during a test run.
-MaxLibrariesPerSite Limit the number of libraries checked per site.
-MaxItemsPerLibrary Limit the number of files and folders checked per library.
-RecoveryOnly Only finish the cleanup of a previous interrupted run.
